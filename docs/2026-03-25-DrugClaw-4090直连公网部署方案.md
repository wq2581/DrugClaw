# DrugClaw 4090 直连公网部署方案

> 日期：2026-03-25  
> 适用场景：不购买云主机，直接使用当前这台 4090 机器对外提供 `drugclaw.site` 服务。  
> 核心原则：不改 DrugClaw 主查询链路，只补 DNS、路由器转发、`systemd`、`nginx`、HTTPS 这几层基础设施。

---

## 1. 当前环境结论

基于当前机器的实际检查结果，这台机器的状态如下：

- 主机名：`bob`
- 系统：`Ubuntu 22.04.5 LTS`
- 项目路径：`/data/boom/Agent/DrugClaw`
- 当前登录用户：`bob`
- 本机局域网 IP：`10.119.191.233`
- 当前外网出口 IP：`202.175.67.248`
- 当前 `drugclaw.site` 解析结果：`208.91.197.27`
- 当前 `80 / 443 / 8000` 端口：未被占用
- 当前机器已验证：`drugclaw serve` 能启动，`/health` 与真实 `/api/query` 可正常返回

这意味着：

1. DrugClaw 服务本身可以在这台机器上运行
2. 但域名 `drugclaw.site` 还没有指向这台机器所在网络
3. 这台机器在内网 `10.x.x.x` 下，说明公网访问还需要依赖路由器端口转发

## 2. 最终目标拓扑

目标拓扑如下：

```text
Browser
  -> https://drugclaw.site
  -> 家庭/办公室路由器公网 IP: 202.175.67.248
  -> 端口转发 80/443 -> 10.119.191.233
  -> nginx
  -> 127.0.0.1:8000
  -> python -m drugclaw serve
  -> upstream LLM / external resource APIs
```

这里最关键的约束是：

- `DrugClaw` 仍然只监听 `127.0.0.1:8000`
- 对公网暴露的只有 `nginx` 的 `80/443`
- 不要把 `8000` 直接开放到外网

## 3. 上线前必须先完成的两件事

如果下面两件事没有完成，后面的所有系统命令都不能让公网用户真正访问到服务。

### 3.1 把 `drugclaw.site` 解析到当前公网出口 IP

在域名控制台里修改 DNS：

- 主机记录：`@`
- 类型：`A`
- 值：`202.175.67.248`

如果你还希望支持 `www.drugclaw.site`，再加一条：

- 主机记录：`www`
- 类型：`CNAME`
- 值：`drugclaw.site`

修改后本机检查：

```bash
getent hosts drugclaw.site
```

正确结果应当包含：

```text
202.175.67.248
```

### 3.2 在路由器做端口转发

你需要在当前网络出口路由器上添加转发规则：

- TCP `80` -> `10.119.191.233:80`
- TCP `443` -> `10.119.191.233:443`

不要添加：

- `8000 -> 10.119.191.233:8000`

如果你的运营商屏蔽 `80/443`，或者你没有路由器管理权限，这条方案就无法真正落地。

## 6. 安装系统组件

下面这些命令需要 `sudo` 密码。

```bash
sudo apt-get update
sudo apt-get install -y nginx certbot python3-certbot-nginx
```

安装完成后检查：

```bash
nginx -v
certbot --version
```

---

## 7. 安装 systemd 服务

项目里已经有模板：

- `deploy/systemd/drugclaw.service`

这份模板当前已经基本匹配此机器：

- 用户：`bob`
- 路径：`/data/boom/Agent/DrugClaw`
- 监听：`127.0.0.1:8000`

安装命令：

```bash
sudo cp /data/boom/Agent/DrugClaw/deploy/systemd/drugclaw.service /etc/systemd/system/drugclaw.service
sudo systemctl daemon-reload
sudo systemctl enable drugclaw
sudo systemctl start drugclaw
sudo systemctl status drugclaw --no-pager
```

查看日志：

```bash
sudo journalctl -u drugclaw -f
```

---

## 8. 安装 nginx 配置

项目里已经有模板：

- `deploy/nginx/drugclaw.conf`

由于模板里还是占位域名，建议直接用替换命令生成线上配置：

```bash
sed 's/your.domain.com/drugclaw.site/g' /data/boom/Agent/DrugClaw/deploy/nginx/drugclaw.conf | sudo tee /etc/nginx/sites-available/drugclaw > /dev/null
```

启用站点：

```bash
sudo ln -sf /etc/nginx/sites-available/drugclaw /etc/nginx/sites-enabled/drugclaw
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl restart nginx
sudo systemctl status nginx --no-pager
```

## 9. 防火墙建议

如果你使用 `ufw`，建议配置为：

```bash
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw deny 8000/tcp
sudo ufw enable
sudo ufw status
```

这样公网只会看到：

- `22`
- `80`
- `443`

不会看到：

- `8000`

## 10. 申请 HTTPS 证书

这一阶段必须满足两个条件：

1. `drugclaw.site` 已经解析到 `202.175.67.248`
2. 路由器已经把 `80/443` 转发到 `10.119.191.233`

先检查：

```bash
getent hosts drugclaw.site
curl -4 https://ifconfig.me
```

两者应当同时体现 `202.175.67.248`。

然后执行：

```bash
sudo certbot --nginx -d drugclaw.site
```

如果还配置了 `www`：

```bash
sudo certbot --nginx -d drugclaw.site -d www.drugclaw.site
```

最终目标地址：

```text
https://drugclaw.site/
```

## 13. 常见失败点

### 13.1 域名没生效

现象：

- `getent hosts drugclaw.site` 还返回旧 IP，例如 `208.91.197.27`

处理：

- 等 DNS 生效
- 确认域名控制台里 `A` 记录真的改成了 `202.175.67.248`

### 13.2 路由器没转发成功

现象：

- 本机 `curl http://127.0.0.1:8000/health` 正常
- 但外部访问 `http://drugclaw.site` 超时

处理：

- 检查路由器 `80/443 -> 10.119.191.233`
- 检查 ISP 是否封锁 `80/443`

### 13.3 证书申请失败

现象：

- `certbot` 无法完成 HTTP challenge

处理：

- 确认 `80` 已经能从公网打到本机 `nginx`
- 确认 DNS 已切换完成

### 13.4 服务能起但系统启动后失效

现象：

- 手工运行 `python -m drugclaw serve` 正常
- `systemd` 启动失败

处理：

- 检查 `User`
- 检查 `WorkingDirectory`
- 检查 `.venv` 路径
- 检查 `navigator_api_keys.json` 是否可读

## 14. 当前方案的边界

这套方案可以让 `drugclaw.site` 对公网可访问，但它仍然是第一版、保守型、单机方案。

当前不包含：

- 用户登录体系
- 细粒度 token 配额
- Redis/消息队列
- 多实例扩容
- 专门的防刷网关

所以更适合：

- 个人项目上线
- 小范围邀请试用
- 简历 / 面试展示
- 初期 Demo 服务

如果后续公网流量明显增大，再考虑加：

- 边缘 WAF / CDN
- 基础鉴权
- 请求限速
- 独立公网入口机

## 15. 一句话执行顺序

真实执行顺序只要记住下面这条：

1. 改 DNS 到 `202.175.67.248`
2. 路由器转发 `80/443 -> 10.119.191.233`
3. 本机确认 `drugclaw serve` 可跑
4. 安装 `nginx + certbot`
5. 配 `systemd`
6. 配 `nginx`
7. 申请 HTTPS
8. 用 `https://drugclaw.site` 验证真实查询
