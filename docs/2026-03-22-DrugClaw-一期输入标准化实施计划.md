# DrugClaw 一期输入标准化实施计划

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 DrugClaw 增加稳定的“药物标准名 + 常见别名”输入标准化能力，让标准名和别名尽量归到同一个 canonical drug，并在固定模式与资源下产出一致结果。

**Architecture:** 在 `DrugClawSystem.query()` 入口增加一层轻量输入标准化，将原始 query 解析为 canonical drug、别名候选和 resolution trace，再由 planner / retriever 优先消费 canonical entities。实现分成三层：alias source、drug name normalizer、workflow integration，并通过单元测试、系统测试和人工回归测试分层验证。

**Tech Stack:** Python, pytest, dataclasses, existing DrugClaw agents/models/query logger, no new heavy runtime dependency in phase 1

## Execution Notes

- Keep the alias seed intentionally small in phase 1. Do not try to solve all drug normalization cases yet.
- Do not add heavy external dependencies for normalization in phase 1.
- If a canonicalization decision is not high-confidence, prefer `unresolved` over forced replacement.
- Preserve backward compatibility for callers that only inspect `result["query"]` and `result["answer"]`.
- Do not remove or rewrite `EntityResolver`; only narrow its role to retrieval-side expansion.

## Exit Criteria

This plan is complete when all of the following are true:

- Standard drug names and common aliases map to the same canonical drug in the new input resolution layer.
- `DrugClawSystem.query()` exposes and logs the normalization trace.
- Planner and retriever prefer canonical drug entities when available.
- Existing relevant tests still pass.
- Manual alias regression produces consistent outputs for the initial drug pairs.
