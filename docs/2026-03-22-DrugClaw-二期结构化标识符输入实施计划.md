# DrugClaw 二期结构化标识符输入实施计划

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 DrugClaw 增加稳定的 `ChEMBL ID + PubChem CID + InChIKey` 入口解析能力，并将其统一归一到 canonical drug name，再复用现有药名/别名标准化链路，确保药名、别名和结构化标识符查询尽量得到一致结果。

**Architecture:** 在 `DrugClawSystem.query()` 入口新增结构化标识符解析层。实现分成三层：identifier source、structured input resolver、workflow integration。入口先做规则识别和外部解析，再把得到的 canonical drug name 交给现有 `DrugNameNormalizer`。失败时保留原始 query 并写入完整 trace，不中断主链路。

**Tech Stack:** Python, pytest, stdlib HTTP client or existing lightweight REST access, dataclasses, current DrugClaw query logging and state model, no heavy chemistry dependency in this phase
