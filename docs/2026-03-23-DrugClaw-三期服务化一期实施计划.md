# DrugClaw 三期服务化一期实施计划

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不破坏现有 CLI 主链路的前提下，为 DrugClaw 增加一个最小可用的 Web/API 服务层和聊天页面，使其可以长期运行在服务器上，并具备基础防崩保护。

**Architecture:** 新增服务层而不重写主编排逻辑。HTTP 请求进入 FastAPI app 后，先经过请求校验、并发控制、超时与错误包装，再调用现有 `DrugClawSystem.query()`。前端采用仓库内静态聊天页，通过 `/api/query` 获取结果；历史查询和 Markdown 报告继续复用现有 `query_logs`。

**Tech Stack:** Python, FastAPI, uvicorn, existing DrugClaw core system, static HTML/CSS/JS, pytest, requests

## Notes for Execution

- 第一期服务化不实现复杂 token 配额
- 先依赖上游账号本身的额度限制
- 重点是不要破坏现有 CLI 主链路
- 所有服务层变更都必须有核心回归测试兜底
- 默认模式要保守，优先 `simple`
